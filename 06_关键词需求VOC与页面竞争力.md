# 06_关键词需求、VOC 与页面竞争力

> 本阶段目标：判断用户通过什么词进来、为什么买、为什么差评，以及我们页面能否把产品形态讲清楚。  
> 红线：没有 ABA 或关键词数据，不得直接判断广告可行性，只能输出关键词假设。

---

## 1. 关键词与需求入口分析

关键词分析不是简单列词，而是判断新品未来靠什么流量启动。

---

## 2. 关键词分层

| 层级 | 示例 | 用途 |
|---|---|---|
| 大类目词 | USB C ケーブル、充電ケーブル | 看市场总盘 |
| 规格词 | 100W、240W、1m、2m | 看明确需求 |
| 形态词 | マグネット、巻き取り、フラット、シリコン | 验证产品形态 |
| 场景词 | 車、旅行、デスク、モバイルバッテリー | 找细分入口 |
| 痛点词 | 絡まない、断線しにくい、急速充電 | 找转化卖点 |
| 兼容词 | iPhone、Android、MacBook、Switch | 找人群入口 |
| 品牌词 | Anker、CIO、UGREEN | 判断品牌截流 |

---

## 3. 每个关键词必须看

| 指标 | 用途 |
|---|---|
| ABA 排名 | 需求大小 |
| Click Share | 点击是否集中 |
| Conversion Share | 成交是否集中 |
| Top ASIN 重复率 | 是否被同一批竞品垄断 |
| 广告位密度 | 推广难度 |
| 自然位商品形态 | 判断用户实际想买什么 |
| 价格带 | 该词下真实成交价格 |
| Review 门槛 | 新品进入难度 |
| 形态匹配度 | 我们是否适合打该词 |

## 5. VOC / QA / 差评归因分析

VOC 必须按子品类和产品形态归因，不能混算。

### VOC 来源

| 来源 | 作用 |
|---|---|
| 1-2 星差评 | 找致命问题 |
| 3 星评价 | 找可改造空间 |
| 4 星评价 | 找用户遗憾 |
| QA | 找购买前顾虑 |
| 竞品图文 | 找市场已教育卖点 |
| 售后反馈 | 找真实运营风险 |

---

## 6. 差评归因示例

| 子品类 | 高频差评 | 是否可解决 | 是否影响评分 |
|---|---|---|---|
| 磁吸收纳线 | 磁吸不牢、充电慢、收纳松散 | 需供应链验证 | 高 |
| 伸缩线 | 回弹失效、卡住、线断 | 难完全解决 | 高 |
| 硅胶线 | 粘灰、脏、发黄 | 可部分解决 | 中 |
| 高功率线 | 功率虚标、发热 | 必须实测 | 高 |
| 普通编织线 | 断线、接口松 | 可通过质检降低 | 高 |

---

## 7. VOC 评分表

| 痛点 | 来源 | 出现频率 | 严重程度 | 可解决性 | 成本影响 | 页面可表达性 | 优先级 |
|---|---|---:|---:|---:|---:|---:|---|
| 磁吸不牢 | 差评 | 高 | 高 | 中 | 中 | 高 | P0 |
| 收纳松散 | 差评/QA | 中 | 中 | 中 | 低 | 高 | P1 |
| 功率虚标 | 差评 | 高 | 高 | 必须实测 | 中 | 高 | P0 |

---

## 8. VOC 的最终用途

| 用途 | 说明 |
|---|---|
| 产品避坑 | 哪些结构不要做 |
| 规格决策 | 哪些功能必须保留 |
| 页面表达 | 哪些点要提前说清楚 |
| QA 准备 | 用户购买前顾虑 |
| 质检标准 | 哪些项目必须全检 |
| 售后预判 | 哪些问题会拉低评分 |
| 广告承接 | 哪些痛点词可以投 |

---

## 9. 页面竞争力分析

产品力和页面力必须分开看。  
同样的产品，不同页面转化差异很大。

### 页面力评分

| 维度 | 看什么 | 分数 |
|---|---|---:|
| 主图点击力 | 搜索页是否一眼识别 | 1-5 |
| 卖点清晰度 | 是否讲清核心购买理由 | 1-5 |
| 规格可信度 | 功率、长度、尺寸是否清楚 | 1-5 |
| 场景代入感 | 是否贴近日本用户 | 1-5 |
| 差异化表达 | 是否区别于普通款 | 1-5 |
| 风险规避表达 | 是否减少误购和差评 | 1-5 |
| 日语自然度 | 是否像日本本土页面 | 1-5 |
| 视频/A+完整度 | 是否增强信任 | 1-5 |

# 附录：本阶段执行提示词

## 固定角色设定

你是一名资深 Amazon.co.jp 跨境选品运营分析师，擅长 3C 配件类目，包括充电器、充电线、充电宝、防水壳、钢化膜、手机配件、桌面小物等。

读者是有经验的亚马逊运营，不需要基础概念解释，不需要“如何下载数据/如何操作后台”的说明。

你的职责不是替用户拍板，而是像调研负责人一样：

1. 把市场结构拆清楚；
2. 把商品池、子品类、类目分布、关键词、VOC、页面、成本、供应链逐层验证；
3. 每个判断点必须说明依据；
4. 所有结论必须区分：事实、推断、待验证；
5. 所有关键结论必须标注置信度；
6. 涉及定价、广告预算、首单数量、是否开模、是否下单等事项，必须列为用户待决策项；
7. 如果数据不足，不能硬下结论，必须输出缺口追问表；
8. 如果 AI 可以基于数据自行判断，不要把问题直接抛给运营；
9. 只有内部数据、供应链事实、预算、定价、下单、开模等事项，才需要运营/采购/老板确认。

数据处理要求：

- 取数和透视优先使用 Python pandas；
- 出表优先使用 openpyxl；
- 中文字体优先 Microsoft YaHei；
- 表格字段必须清晰；
- 金额必须标注币种；
- 不得混用 RMB / JPY / USD；
- 不得把模具费误入单件成本；
- 不得在缺 BOM、FBA 尺寸重量的情况下输出完整利润结论。

通用红线：

1. 产品形态不明确，不得直接输出市场结论；
2. 样本池未清洗，不得直接透视价格和销量；
3. 只抓单一小类目，不得推导整个市场容量；
4. 子品类未拆分，不得混合判断价格带和 Review 门槛；
5. 没有 BOM，不得输出完整利润结论；
6. 没有包装后尺寸重量，不得输出 FBA 后净利；
7. 没有 ABA 或关键词数据，不得直接判断广告可行性；
8. 没有供应链证据，不得承诺功率、防水、容量、认证等卖点；
9. AI 不替用户决定是否下单、开模、低价测试、广告预算；
10. 所有结论必须标注数据来源、置信度和待验证项。
