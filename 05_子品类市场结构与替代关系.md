# 05_子品类市场结构与替代关系

> 本阶段目标：对已拆分的子品类分别做市场结构分析，并判断目标产品与竞品之间的替代关系。  
> 红线：不同子品类不能混合判断价格带、Review 门槛和广告压力。

---

## 1. 子品类市场结构分析目标

子品类分析要回答：

```text
哪个子品类有容量？
哪个子品类有新品机会？
哪个子品类价格带能支撑利润？
哪个子品类被大品牌垄断？
哪个子品类只是看起来热，但实际不适合我们？
```

---

## 2. 每个子品类必须单独看

| 维度 | 指标 |
|---|---|
| 容量 | Top 20 月销量、月销售额 |
| 价格带 | 到手价 P25 / P50 / P75 |
| 头部集中度 | Top 5 / Top 10 销量占比 |
| Review 门槛 | Review P50、Rating 分布 |
| 新品成功率 | 近 180 天新品中月销破 100 / 300 / 500 的数量 |
| 品牌垄断 | 大品牌销量占比 |
| 广告压力 | 广告位密度、广告 ASIN 重复率 |
| 父体依赖 | 是否靠父体评价共享 |
| 价格战程度 | 低价款占比、Coupon 密度 |
| 页面成熟度 | 主图、A+、视频是否高度同质化 |

## 4. 新品成功样本判断

新品成功样本不是“上架时间短”就算。  
建议至少同时满足以下条件之一：

| 条件 | 含义 |
|---|---|
| 上架 ≤ 180 天，月销 ≥ 100 | 有初步动销 |
| 上架 ≤ 180 天，月销 ≥ 300 | 有可参考爬坡能力 |
| 上架 ≤ 365 天，Review 低但销量稳定 | 可能具备新品切入路径 |
| 低 Review 但自然位较好 | 页面或形态有优势 |
| 无大品牌背书但销量稳定 | 说明非品牌玩家仍有机会 |

### 新品样本字段

| 字段 | 说明 |
|---|---|
| ASIN | 新品 ASIN |
| 上架时间 | 判断新品周期 |
| 月销量 | 动销能力 |
| Review 数 | 评价门槛 |
| Rating | 质量反馈 |
| 子品类 | 所属子品类 |
| 价格 | 到手价 |
| 核心卖点 | 新品靠什么切入 |
| 流量来源判断 | 自然 / 广告 / 促销 / 父体 |

---

## 5. 竞争集中度

### 头部集中度

```text
Top5 销量占比 = Top5 月销量 / 子品类总月销量
Top10 销量占比 = Top10 月销量 / 子品类总月销量
```

| 集中度 | 判断 |
|---|---|
| 高 | 头部垄断明显，新品难切 |
| 中 | 有腰部竞争空间 |
| 低 | 市场分散，可寻找细分机会 |

---

## 6. 产品形态匹配度分析

目标：判断我们想做的形态是否是市场真实需求。

| 维度 | 判断问题 |
|---|---|
| 市场存在感 | 同形态商品数量是否足够 |
| 销量表现 | 同形态商品是否能稳定出单 |
| 价格溢价 | 是否比普通款有溢价 |
| 评论反馈 | 用户是否认可这个形态 |
| 差评风险 | 是否引入新问题 |
| 页面表达 | 主图能否一眼讲清楚 |
| 搜索需求 | 是否有对应关键词 |
| 供应链难度 | 结构是否复杂 |
| 质量稳定性 | 是否容易拉低评分 |

### 形态匹配度表

| 产品形态 | 市场定位 | 同形态ASIN数 | 月销量表现 | 是否可溢价 | 主要风险 | 初步结论 |
|---|---|---:|---|---|---|---|

## 7. 替代关系分析

子品类相近不等于直接竞争。  
必须判断目标产品和竞品之间的替代关系。

| 类型 | 含义 | 策略 |
|---|---|---|
| 直接替代 | 规格、形态、价格高度接近 | 比价格、页面、评论、供应链 |
| 功能替代 | 解决同一痛点，但形态不同 | 强调痛点和场景 |
| 场景替代 | 特定场景下用户二选一 | 打场景词和场景图 |
| 非替代 | 看似同类，但购买理由不同 | 不放进主竞争池 |

### 充电线示例

| 对象 | 替代关系 | 处理方式 |
|---|---|---|
| 普通 C-C 线 | 弱替代 | 不直接比最低价 |
| 伸缩卷线线 | 场景替代 | 关注收纳/车载场景 |
| 硅胶不缠绕线 | 痛点替代 | 关注不缠绕需求 |
| CIO フラットスパイラルケーブル | 直接替代 | 深度拆解 |
| 100W 编织线 | 规格替代 | 只在高功率需求下对比 |

---

## 8. 竞品分层

| 层级 | 说明 | 研究重点 |
|---|---|---|
| 头部品牌型 | Anker、UGREEN、CIO、ELECOM 等 | 价格天花板、信任壁垒 |
| 性价比走量型 | 中低价、高销量、强促销 | 真实成交价、利润压力 |
| 差异化功能型 | 形态或功能明显 | 是否可复制 |
| 新品爬坡型 | 上架时间短但增长快 | 新品启动路径 |
| 低价尾货型 | 极低价、质量弱 | 是否拉低价格带 |
| 自营/VC 型 | Amazon 自营或大品牌供货 | 平台资源倾斜 |

# 附录：本阶段执行提示词

## 固定角色设定

你是一名资深 Amazon.co.jp 跨境选品运营分析师，擅长 3C 配件类目，包括充电器、充电线、充电宝、防水壳、钢化膜、手机配件、桌面小物等。

读者是有经验的亚马逊运营，不需要基础概念解释，不需要“如何下载数据/如何操作后台”的说明。

你的职责不是替用户拍板，而是像调研负责人一样：

1. 把市场结构拆清楚；
2. 把商品池、子品类、类目分布、关键词、VOC、页面、成本、供应链逐层验证；
3. 每个判断点必须说明依据；
4. 所有结论必须区分：事实、推断、待验证；
5. 所有关键结论必须标注置信度；
6. 涉及定价、广告预算、首单数量、是否开模、是否下单等事项，必须列为用户待决策项；
7. 如果数据不足，不能硬下结论，必须输出缺口追问表；
8. 如果 AI 可以基于数据自行判断，不要把问题直接抛给运营；
9. 只有内部数据、供应链事实、预算、定价、下单、开模等事项，才需要运营/采购/老板确认。

数据处理要求：

- 取数和透视优先使用 Python pandas；
- 出表优先使用 openpyxl；
- 中文字体优先 Microsoft YaHei；
- 表格字段必须清晰；
- 金额必须标注币种；
- 不得混用 RMB / JPY / USD；
- 不得把模具费误入单件成本；
- 不得在缺 BOM、FBA 尺寸重量的情况下输出完整利润结论。

通用红线：

1. 产品形态不明确，不得直接输出市场结论；
2. 样本池未清洗，不得直接透视价格和销量；
3. 只抓单一小类目，不得推导整个市场容量；
4. 子品类未拆分，不得混合判断价格带和 Review 门槛；
5. 没有 BOM，不得输出完整利润结论；
6. 没有包装后尺寸重量，不得输出 FBA 后净利；
7. 没有 ABA 或关键词数据，不得直接判断广告可行性；
8. 没有供应链证据，不得承诺功率、防水、容量、认证等卖点；
9. AI 不替用户决定是否下单、开模、低价测试、广告预算；
10. 所有结论必须标注数据来源、置信度和待验证项。
