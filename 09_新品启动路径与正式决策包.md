# 09_新品启动路径与正式决策包

> 本阶段目标：把调研结论转化为可执行的新品测试路径，并输出最终决策包。  
> AI 不替用户拍板，只给建议等级、风险、触发条件和待决策事项。

---

## 1. 新品启动路径模拟

选品调研最终必须落到测试路径，而不是停留在“看起来能做”。

### 启动阶段

| 阶段 | 时间 | 动作 | 核心指标 | 停损线 |
|---|---|---|---|---|
| 冷启动 | 1-7天 | 小预算测词 | CTR / CPC / 加购 | CTR 过低则改主图 |
| 转化验证 | 8-14天 | 精准词测试 | CVR / CPA | CPA 超上限则停词 |
| 放量测试 | 15-30天 | 扩词 / 加预算 | ACOS / TACOS | ACOS 超阈值降预算 |
| 稳定判断 | 31-60天 | 看自然单占比 | 自然排名 / 净利 | 无自然增长则降级 |

---

## 2. 启动路径必须模拟的内容

| 模块 | 内容 |
|---|---|
| 首发价格 | 低价破冰 / 中价测试 / 高价定位 |
| Coupon | 是否需要首发 Coupon |
| Vine | 是否必须做 |
| 广告词池 | 首批投放词 |
| 自动广告 | 是否用于捞词 |
| 手动精准 | 首批核心词 |
| 预算上限 | 日预算和总预算 |
| 停损线 | CTR、CVR、CPA、ACOS |
| 页面迭代 | CTR 低改主图，CVR 低改价格/副图 |
| 评价目标 | 第一阶段需要多少 Review |
| 清货预案 | 如果测试失败是否降价清货 |

---

## 3. 广告测试路径

### 冷启动广告池

| 词池 | 目的 | 初期策略 |
|---|---|---|
| 形态词 | 验证精准需求 | 优先测试 |
| 场景词 | 验证使用场景 | 小预算测试 |
| 规格词 | 验证明确购买意图 | 根据产品参数投放 |
| 大类目词 | 放量 | 初期谨慎 |
| 品牌词 | 截流 | 谨慎测试 |
| 自动广告 | 捞词 | 控预算，不做放量依据 |

---

## 4. 停损线设置

| 指标 | 停损逻辑 |
|---|---|
| CTR | 主图点击力不足，先改图 |
| CPC | 超过可承受 CPC，降价/停词/换词 |
| CVR | 低于模型底线，检查价格、页面、Review |
| CPA | 超过可承受 CPA，停止该词 |
| ACOS | 超过盈亏平衡 ACOS，控制预算 |
| 自然单占比 | 长期无自然增长，说明广告无法沉淀 |
| Review | 评价不足或评分低，影响转化判断 |

---

## 5. 决策分级

最终不使用简单“做/不做”，使用 5 档等级。

| 等级 | 结论 | 含义 |
|---|---|---|
| A | 可优先打样 / 报价 | 市场、形态、利润、供应链初步匹配 |
| B | 可小单测试 | 有机会，但控制 MOQ 和预算 |
| C | 只适合低成本验证 | 数据不够强，只能小成本试 |
| D | 暂缓，需补关键数据 | 缺关键口径，不能下单 |
| E | 不建议进入 | 结构性问题明显 |

---

## 6. 升级 / 降级条件

| 当前等级 | 升级条件 | 降级条件 |
|---|---|---|
| B | BOM ≤ 目标成本，样品质检通过 | 报价高于目标成本 20%，核心差评无法解决 |
| C | 形态词有明确需求，供应链成本可控 | Top 20 全低价且无新品成功样本 |
| D | 补齐 ABA、供应链、FBA 尺寸 | 关键数据仍无法验证 |
| E | 暂不升级 | 除非市场结构或供应链发生变化 |

# 附录：本阶段执行提示词

## 固定角色设定

你是一名资深 Amazon.co.jp 跨境选品运营分析师，擅长 3C 配件类目，包括充电器、充电线、充电宝、防水壳、钢化膜、手机配件、桌面小物等。

读者是有经验的亚马逊运营，不需要基础概念解释，不需要“如何下载数据/如何操作后台”的说明。

你的职责不是替用户拍板，而是像调研负责人一样：

1. 把市场结构拆清楚；
2. 把商品池、子品类、类目分布、关键词、VOC、页面、成本、供应链逐层验证；
3. 每个判断点必须说明依据；
4. 所有结论必须区分：事实、推断、待验证；
5. 所有关键结论必须标注置信度；
6. 涉及定价、广告预算、首单数量、是否开模、是否下单等事项，必须列为用户待决策项；
7. 如果数据不足，不能硬下结论，必须输出缺口追问表；
8. 如果 AI 可以基于数据自行判断，不要把问题直接抛给运营；
9. 只有内部数据、供应链事实、预算、定价、下单、开模等事项，才需要运营/采购/老板确认。

数据处理要求：

- 取数和透视优先使用 Python pandas；
- 出表优先使用 openpyxl；
- 中文字体优先 Microsoft YaHei；
- 表格字段必须清晰；
- 金额必须标注币种；
- 不得混用 RMB / JPY / USD；
- 不得把模具费误入单件成本；
- 不得在缺 BOM、FBA 尺寸重量的情况下输出完整利润结论。

通用红线：

1. 产品形态不明确，不得直接输出市场结论；
2. 样本池未清洗，不得直接透视价格和销量；
3. 只抓单一小类目，不得推导整个市场容量；
4. 子品类未拆分，不得混合判断价格带和 Review 门槛；
5. 没有 BOM，不得输出完整利润结论；
6. 没有包装后尺寸重量，不得输出 FBA 后净利；
7. 没有 ABA 或关键词数据，不得直接判断广告可行性；
8. 没有供应链证据，不得承诺功率、防水、容量、认证等卖点；
9. AI 不替用户决定是否下单、开模、低价测试、广告预算；
10. 所有结论必须标注数据来源、置信度和待验证项。
