# 00_项目入口与总纲

> 适用场景：Amazon.co.jp 跨境新品调研，尤其适用于 3C 配件类目：充电器、充电线、充电宝、防水壳、钢化膜、手机配件、桌面小物等。  
> 使用目的：把“选品调研”拆成可复用、可断点续传、可由 AI + 运营协作执行的标准流程。

---

## 1. SOP 核心定位

本 SOP 不是让 AI 直接判断“这个产品能不能做”，而是建立一套完整的调研系统：

```text
确认产品形态
→ 构建市场商品池
→ 校验跨小类目分布
→ 清洗样本
→ 拆分子品类
→ 分析市场结构
→ 验证关键词需求
→ 归因 VOC
→ 计算利润与广告承受力
→ 验证供应链
→ 输出新品启动路径与决策包
```

AI 的职责：

- 把市场拆清楚；
- 把数据算准确；
- 把风险列完整；
- 把事实、推断、待验证项分开；
- 给出建议等级、触发条件和停损线。

运营/采购/老板的职责：

- 运营确认业务方向、竞品池、关键词、页面策略；
- 采购补充 BOM、MOQ、交期、认证、包装后尺寸重量；
- 老板/用户拍板资金、定价、首单数量、广告预算、是否开模、是否低价测试。

## 2. 完整阶段结构

| 阶段 | 文档 | 核心作用 |
|---|---|---|
| 00 | 项目入口与总纲 | 定义整体规则、红线、阶段结构 |
| 01 | 调研入口定义与AI判断边界 | 明确调研类型、AI自判/运营确认/用户拍板边界 |
| 02 | 产品形态假设确认 | 先锁定产品形态，再进入市场分析 |
| 03 | 市场商品数据采集与样本清洗 | 构建商品池，清洗无效样本 |
| 04 | 类目分布与子品类拆分 | 解决同品跨小类目问题，并拆分子品类 |
| 05 | 子品类市场结构与替代关系 | 判断哪个子品类值得打，谁是真正竞品 |
| 06 | 关键词需求、VOC与页面竞争力 | 判断用户入口、产品痛点和页面转化空间 |
| 07 | 成本利润与广告反推模型 | 判断价格、成本、广告是否跑得通 |
| 08 | 供应链验证 | 验证供应链是否能做到页面承诺 |
| 09 | 新品启动路径与正式决策包 | 输出测试打法、决策等级、待拍板事项 |
| 10 | AI缺口追问与阶段状态管理 | 管理缺数据、置信度、退回/终止 |

## 4. 关键红线

以下规则必须写死，不能被后续分析绕过。

| 红线 | 说明 |
|---|---|
| 产品形态不明确，不得直接输出市场结论 | 否则市场样本没有分析对象 |
| 样本池未清洗，不得直接透视价格和销量 | 否则价格带、销量、Review 门槛失真 |
| 只抓单一小类目，不得推导整个市场容量 | 同品可能分布在多个小类目 |
| 子品类未拆分，不得混合判断价格带和 Review 门槛 | 普通款、磁吸款、伸缩款逻辑不同 |
| 没有 BOM，不得输出完整利润结论 | 只能输出市场侧判断 |
| 没有包装后尺寸重量，不得输出 FBA 后净利 | FBA 费用会直接影响利润 |
| 没有 ABA 或关键词数据，不得直接判断广告可行性 | 只能输出关键词假设 |
| 没有供应链证据，不得承诺功率、防水、容量、认证等卖点 | 防止页面虚标和合规风险 |
| AI 不替用户决定是否下单、开模、打低价、投入广告预算 | 只能给建议、风险和触发条件 |
| 所有结论必须标注事实、推断、待验证 | 防止把推断写成结论 |

---

## 5. 阶段输出统一格式

每个阶段结束必须输出 5 个区块：

```text
1. AI 已自行判断
2. 判断依据
3. 当前结论置信度
4. 需要运营确认
5. 需要用户/老板拍板
```

---

## 6. 事实 / 推断 / 待验证

所有关键结论必须分三层：

| 类型 | 示例 |
|---|---|
| 事实 | Top 20 到手价 P50 = ¥1,980 |
| 推断 | 该子品类用户可接受价格集中在 ¥1,780-2,480 |
| 待验证 | 我们的 BOM 是否能支撑 ¥1,980 到手价 |

---

## 7. 置信度规则

| 置信度 | 定义 |
|---|---|
| 高 | 多来源一致，字段完整，可用于阶段结论 |
| 中 | 数据基本可用，但部分字段估算或缺失 |
| 低 | 数据缺失较多，只能用于方向判断 |

---

## 8. 阶段状态

| 状态 | 处理 |
|---|---|
| 通过 | 进入下一阶段 |
| 退回 | 补数据、重拆子品类或重新清洗样本 |
| 终止 | 结构性问题明显，不继续投入调研资源 |

---

## 9. Python 出表要求

调研数据处理与出表优先使用：

```text
pandas：取数、清洗、透视、计算
openpyxl：多 Sheet 输出、格式化、标色、冻结首行、筛选
```

格式要求：

- 中文字体优先 Microsoft YaHei；
- 表头加粗；
- 冻结首行；
- 自动筛选；
- 金额格式统一；
- 高风险项标色；
- 待补数据标色；
- 所有金额字段明确币种：JPY / RMB / USD。

# 附录：通用执行提示词使用方式

每次执行某个阶段时，提示词由 4 部分组成：

```text
1. 固定角色设定
2. 当前项目状态
3. 当前阶段任务
4. 输出格式与判断门
```

AI 不得一次性跳过多个阶段。  
每个阶段结束必须输出：

```text
1. AI 已自行判断
2. 判断依据
3. 当前结论置信度
4. 需要运营确认
5. 需要采购/供应商补数
6. 需要用户/老板拍板
7. 阶段状态：通过 / 退回 / 终止
```

## 建议执行方式

```text
第 1 次：固定角色设定 + 01 阶段提示词
第 2 次：固定角色设定 + 项目状态 + 02 阶段提示词
第 3 次：固定角色设定 + 项目状态 + 上阶段输出 + 03 阶段提示词
……
```

这样 AI 不会跳阶段，也不会把没验证的数据直接写成结论。
